Выберите товар и оба периода

1. Скачать шаблон

Шаблон подстраивается под площадку выбранного товара — свой набор полей рекламы для WB/Ozon/Я.Маркет. Заполните строки и сохраните файл

2. Загрузить заполненный файл

Даты, которые уже есть в истории товара, будут перезаписаны данными из файла — остальное добавится как новые дни

3. Исправить «Выкупили»/«Отмены» после старого импорта

Если раньше при импорте пустая колонка «Выкупили» или «Отмены» превращалась в 0 (это было исправлено) — здесь можно задним числом вернуть эти дни в состояние «неизвестно», не перезагружая файл заново. Затронет только дни, где сейчас стоит 0 — дни с другими значениями не тронет.

Если заполнить «Не позже даты» — тронет только дни до этой даты включительно (например, конец импортированного периода), чтобы не задеть настоящие нули, введённые уже вручную после обновления

План и факт

По площадке — общие цифры плана/факта по всем товарам одного маркетплейса сразу.

По товару — план и факт конкретного товара. План вводится вручную раз в месяц (в ₽ и шт.) либо считается кнопкой «Рассчитать автоматически» — по среднему темпу продаж за последние месяцы с поправкой на рост и на остаток товара на складе (с учётом даты поступления новой партии).

Run-rate — прогноз, куда придём к концу месяца, если ничего не менять (факт шт/день × дни в месяце). Показывается только для текущего месяца.

Темп за 7 дней — средняя скорость продаж за последнюю неделю и сравнение с неделей до неё — быстрее реагирует на изменения, чем Run-rate.

Группа плана — для случаев, когда один и тот же товар продаётся с двух и более карточек (частый случай на Я.Маркете). Назначается на вкладке «Товары» кнопкой «Группа плана» — карточки с одинаковым названием группы получают общий план, общий остаток на складе и общее поступление, при этом ежедневные данные, воронка и точки роста у каждой карточки остаются свои.

Если дубли появляются и пропадают периодически (например, полгода одна карточка, потом временно вторая, потом снова одна) — специально ничего откатывать не нужно: группу можно оставить назначенной навсегда. Если вторая карточка перестала использоваться (или удалена), в общий факт группы она просто перестаёт что-либо добавлять — план продолжает корректно считаться по оставшейся карточке. Единственное, о чём стоит помнить: план за месяцы, которые были введены до того, как назначили группу, при первом объединении карточек переносится в общий план группы автоматически — переносить вручную не придётся.

Ежедневные данные

Основной раздел для ежедневного ввода. Данные по воронке вносятся за вчерашний день. Текущая цена и изменения в ЛК — на текущий день.

Цена в ЛК / для покупателя / с Ozon картой — при первом заполнении подставляется прежняя цена автоматически. Если цена изменилась — впишите новую и отметьте тип изменения «Цена»: в этот день сохранится прежняя цена (она и правда действовала в этот день), а новая автоматически появится в черновике на завтра.

Черновик — фиксирует цену и изменения в ЛК на текущий день. На следующий день остаётся дозаполнить показатели воронки, когда будут данные.

Рекламные ставки и Акции — справочная информация, на расчёты не влияет, но переносится изо дня в день автоматически, чтобы зафиксировать текущие значения. Изменение ставки можно зафиксировать с помощью ручного ввода.

Выкупили / Отмены — для «Отмены» по умолчанию подставляется 0, менять не нужно, если это верно; «Выкупили» вводится вручную каждый раз, без дефолта. Для обоих полей: если оставить поле пустым (не 0, а именно пусто) — день сохранится без этого показателя как «неизвестно», а не как ноль. Если показатель не заполняется больше недели подряд — он перестаёт участвовать в «Точках роста» (там появится отдельная пометка), чтобы не считать проценты по устаревшим данным.

Аномалии — сравнивает последний полностью заполненный день с днём перед ним и предупреждает о резких скачках (показы, CTR, конверсии, рекламные расходы).

История по товару — вся история день за днём, с цветовой подсветкой изменений показателей. Строку «Что изменили» можно поправить от руки прямо в таблице.

Воронка и точки роста

Воронка за месяц — показы → клики → корзины → заказы → выкупы, с процентом перехода на каждом этапе.

Диагноз воронки — один общий вывод о том, что сейчас главная проблема (или что всё в порядке), плюс вероятные причины с примерной значимостью каждой. Сравнивает последние 2 недели фактических (не черновых) дней — неделю перед этим с самой последней неделей, а не половины выбранного месяца, поэтому работает даже в начале месяца.

Точки роста — подробный разбор по каждому этапу воронки отдельно: показывается только то, что ниже нормы, с причинами и что проверить. Пороги нормы можно настроить под конкретный товар на вкладке «Товары».

Рекомендации — про темп плана, ДРР и динамику конверсии, с конкретными цифрами.

Журнал изменений — оценивает, помогло ли конкретное изменение карточки: сравнивает нужный показатель до/после, с поправкой на то, сколько дней нужно подождать для честной оценки этого типа правки.

Сравнение периодов

Сравнение показателей товара за два периода — по месяцам или по датам (диапазон не длиннее 31 дня). Показывает, что выросло/упало, и для просевших показателей — гипотезу и рекомендацию.

Импорт данных

Если есть данные за прошлые периоды — можно скачать Excel-шаблон под конкретную площадку, заполнить и загрузить обратно. Даты, которые уже есть в истории, будут перезаписаны файлом, остальные добавятся как новые.
